# Agent 工作原则 / Working Principles

> 项目级持久化规则。任何环境 clone 本仓库后自动继承；本地 `data/profile.json` 的 `work_style.working_principles` 若有值则优先覆盖。
> 由用户显式确立，作为 Hermes Agent 的全局行为约束。

---

## 规则一：从第一性原理出发

**适用场景**: 解决问题、修 BUG、设计架构或方案时。

**要求**:
- 先回到事物的本质约束与基本事实，拆解隐含假设，再推导方案；
- 不从既有做法、惯例、或上游框架的默认行为出发；
- 遇到"通常这么做"时，先问"为什么这么做、约束是什么、换一种做法是否更本质"；
- 优先识别真正的根因，而非症状层打补丁。

## 规则二：复杂任务后多 Agent 对抗性审查

**适用场景**: 完成相对比较复杂的任务后。

**要求**:
- 开启多个独立角色的 Agent 进行对抗性审查；
- 审查 Agent 从反方视角质疑结论、复现验证、寻找边界条件与反例；
- 只有通过对抗性审查后才视为交付；
- 审查发现的问题必须回溯修复，不得带病交付。

---

## 加载机制

- `hermes.profile.load_profile()` 在 `work_style.working_principles` 为空时，自动从本文件解析规则条目回填。
- 本地 `data/profile.json` 一旦写入 `working_principles`，则以本地值为准（允许个性化覆盖）。
- 本文件变更随 git 提交持久化，跨环境共享。
