## 4. 缺陷与改进项

### 4.1 P1（M2 必修）

| ID | 缺陷 | 影响 | 建议 |
|---|---|---|---|
| P1-1 | 检索准确率 83.3% 未达 90% KPI | 用户体验 | M2 用真实 Embedding（bge-small-zh）+ 评测集校准 |
| P1-2 | 低置信阈值 0.005 是经验值 | 误判 | M2 跑评测集确定最佳阈值 |

### 4.2 P2（M2 可修）

| ID | 缺陷 | 建议 |
|---|---|---|
| P2-1 | SSE 流式无 token 用量统计 | M2 增加 token 计数 |
| P2-2 | JWT 单用户，无法区分操作人 | M3 多租户时升级 |
| P2-3 | 无操作审计日志 | M2 增加导入/删除审计 |
| P2-4 | 历史记录无搜索/筛选 | M2 历史搜索 + 标签 |
| P2-5 | 无文档批量导入 | M2 批量上传 + 拖拽 |

### 4.3 P3（M3 考虑）

- 移动端适配
- 监控告警（Prometheus + Grafana）

## 5. 验收结论

### 5.1 整体结论：✅ **通过**

M1 全部 8 项核心任务交付，120/120 测试通过，E2E 验证全部端点正常。M0 路线图"M1 核心质量修复"目标达成。

### 5.2 关键成就

1. **质量基线建立**：120 测试 + 安全机制 + 降级策略，可放心迭代
2. **真实 LLM/Embedding 接入路径打通**：环境变量切换即可上生产
3. **流式体验落地**：首字 < 50ms，引用先于正文
4. **合规基础**：未成年保护 + JWT 认证
5. **可观测**：health 端点暴露 11 项运行态

### 5.3 风险提示

1. **代码工作区被重置**：本验收基于 commit `1b61eea` 的交付状态记录，需恢复代码后回归验证
2. **真实 LLM 未实测**：Provider 抽象已就绪，但未用智谱/OpenAI 实跑，M2 需补充
3. **检索准确率卡 83.3%**：M2 必须用真实 Embedding 突破 90% KPI

### 5.4 上线建议

M1 可作为 **内部试用版** 上线：
- 配置 `KB_AUTH_ENABLED=true` + 强密码
- 配置 `KB_AGE_GATE=true`
- 暂不配置真实 LLM Key（Mock 可用）
- 限制并发用户 < 20

正式商用前需完成 M2 + M3 关键项。
