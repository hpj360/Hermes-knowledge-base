## 1. 文档目的

本文从产品战略层面，对 Hermes KB 的**生态位、目标用户、JTBD、增长飞轮、冷启动路径与能力边界**做深度分析，为 M2 全新前端设计提供 9 项约束映射。教学投影模式与多租户预留**不在本期范围**（已移除）。

## 2. 产品本质

Hermes KB 是一个**私人酒类知识管家**——用户把酒类资料（百科、品鉴笔记、产区法规）灌入本地知识库，通过引用式问答获得可信答案，所有回答带溯源引用并可一键跳转原文。

三句话定位：
- **不是**通用聊天机器人，**是**基于私有文档的引用式问答引擎
- **不是**搜索引擎，**是**知识管家——答案只来自你导入的内容
- **不是**公开百科，**是**个人/小团队的私有酒类知识沉淀

## 3. 生态位分析

### 3.1 竞品光谱

| 类型 | 代表 | 与 Hermes KB 关系 |
|---|---|---|
| 通用 LLM 对话 | ChatGPT / 智谱清言 | 互补——通用对话补盲区，Hermes 补私有溯源 |
| 公开酒类百科 | 维基百科 / 烈酒中文网 | 上游——内容可导入 Hermes 形成私有库 |
| 笔记工具 | Notion / Obsidian | 平行——Hermes 强在"问答式检索"，笔记强在"编辑式组织" |
| 企业 RAG 平台 | Dify / FastGPT | 错位——它们面向通用知识库，Hermes 聚焦酒类垂直 + 引用溯源 |

### 3.2 差异化壁垒

1. **引用溯源硬约束**：每个答案必带 [N] 引用，点击跳转原文 chunk——通用 LLM 做不到
2. **酒类垂直语境**：冷启动种子覆盖六大基酒，分类/标签预设贴合酒类知识结构
3. **私有部署零依赖**：单进程 SQLite，无需云服务，数据不出本地
4. **合规前置**：年龄门 + 单用户认证，符合酒类内容监管

## 4. 目标用户

### 4.1 核心用户：私人酒类爱好者（"知识囤积者"）

- 画像：25-45 岁，有饮酒习惯，主动收集酒类资料，渴望系统化沉淀
- 痛点：资料散落在浏览器书签/截图/聊天记录，需要时找不到、不敢信
- 期望：把资料灌进一个地方，用提问方式快速检索，答案能溯源验证

### 4.2 次级用户：小团队知识共享（酒吧/培训）

- 画像：酒吧主理人、酒类培训师、进口商产品经理
- 痛点：团队酒类知识口径不一，新人培训依赖老人记忆
- 期望：统一知识库 + 引用式问答，新人能自助查询且答案可信

### 4.3 非目标用户（本期排除）

- 大众轻度饮酒者（无系统化沉淀需求）
- 教学投影场景（已移除）
- 多租户 SaaS（已移除，M3 再议）

## 6. 增长飞轮

```
导入更多酒类文档
   ↓
知识库覆盖更广 → 问答命中率提升
   ↓
用户更信任答案 → 更频繁提问
   ↓
提问暴露知识盲区 → 引导补充文档
   ↓
（回到起点）导入更多文档
```

飞轮健康度四维度（仪表盘可视化）：
1. **内容输入**：文档/分片增长率
2. **检索质量**：问答命中率 + 平均置信度
3. **引用密度**：每答案平均引用数
4. **用户活跃**：周提问频次

## 7. 冷启动路径（L0-L4 引导链）

| 阶段 | 触发 | 引导动作 | 目标 |
|---|---|---|---|
| L0 | 首次访问 | 年龄门确认 | 合规准入 |
| L1 | 年龄通过 | 品牌首页（Hero + 头条问答 + 热门文档） | 建立价值认知 |
| L2 | 进入问答页 | 空状态引导 + 3 个示例问题 | 降低首次提问门槛 |
| L3 | 首次问答完成 | 引导"这个答案引用了哪些文档，点进去看看" | 培养溯源习惯 |
| L4 | 文档库为空 | 空状态引导"导入你的第一篇文档" | 启动飞轮 |

## 8. 能力边界（本期不做）

| 边界 | 原因 | 替代/未来 |
|---|---|---|
| 不做多轮对话记忆 | 单轮问答已覆盖核心 JTBD | M3 评估 |
| 不做图像/视频理解 | 文本知识库即可 | M3+ |
| 不做公开知识联网 | 私有库定位，避免污染溯源 | 永不 |
| 不做多租户 | 单用户/小团队场景，降低复杂度 | M3 评估 |
| 不做教学投影 | 场景窄，投入产出低 | 永不 |
| 不做自动爬取导入 | 版权与质量风险 | 手动导入为主 |

## 9. 六条战略设计原则

| # | 原则 | 设计映射 |
|---|---|---|
| P1 | **引用即信任**：每个答案必带可点击溯源引用 | 引用卡片视觉权重 + chunk 高亮跳转 |
| P2 | **知识即资产**：导入门槛要低，沉淀要可见 | 导入多 tab + 文档库 + 仪表盘飞轮健康度 |
| P3 | **垂直即专业**：视觉与内容体现酒类垂直感 | 高级酒类杂志感视觉语言（酒红+暗金+衬线） |
| P4 | **私有即安心**：数据不出本地，合规前置 | 年龄门 + 单用户认证 + 本地部署说明 |
| P5 | **冷启动即友好**：空库也能体验价值 | L0-L4 引导链 + 种子知识 + 示例问题 |
| P6 | **极简即克制**：功能收敛，不堆砌 | 9 页面 + 3 modal，每页单一职责 |

## 10. 视觉语言战略选择（D15）

### 决策 D15：高级酒类杂志感视觉

| 维度 | 选择 | 理由 |
|---|---|---|
| 主色 | 深酒红 `#8B1A36` | 呼应酒类主题，沉稳不轻浮 |
| 强调色 | 暗金 `#C9A227` | 杂志感高级质感，区分于通用 SaaS 蓝绿 |
| 中性色 | 暖灰墨色 | 比纯灰更有温度，贴合"私人管家"气质 |
| 标题字体 | 衬线（Noto Serif SC） | 杂志感、权威感、区别于科技产品无衬线 |
| 正文 | 无衬线（Inter） | 阅读舒适 |

被否决方案：
- 科技蓝绿（太通用，无酒类辨识度）
- 全黑极简（太冷，缺温度）
- 复古手绘（太装饰，影响信息密度）

## 附录 A：六大基酒冷启动种子

金酒、威士忌、葡萄酒、中国白酒、朗姆酒、龙舌兰——覆盖鸡尾酒六大基酒 + 中国特色白酒，保证首次问答即可体验价值。

## 附录 B：术语表

- **chunk**：文档分片，引用溯源的最小单位
- **RRF**：检索结果融合，多路召回排序算法
- **冷启动**：用户首次使用时的引导体验
- **飞轮**：自我强化的增长循环
