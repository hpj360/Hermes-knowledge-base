### 2.2 详细规格

#### M2-01 真实 LLM + Embedding 接入与评测集（W1，P0）

**目标**：用真实模型替换 Mock，建立可量化的评测体系。

**任务**：
1. 接入智谱 GLM-4-Flash（D1 决策）
   - 配置 `KB_LLM_PROVIDER=openai` / `KB_LLM_BASE_URL=https://open.bigmodel.cn/api/paas/v4` / `KB_LLM_MODEL=glm-4-flash`
   - 验证流式 SSE 解析（智谱遵循 OpenAI 协议）
2. 接入 bge-small-zh-v1.5（D2 决策）
   - 配置 `KB_EMBEDDING_PROVIDER=sentence_transformers` / `KB_EMBEDDING_ST_MODEL=BAAI/bge-small-zh-v1.5`
   - 验证维度（512）+ 自动 reindex
3. 构建评测集（50 题）
   - 来源：5 篇种子文档衍生
   - 分布：金酒 8 / 威士忌 10 / 葡萄酒 10 / 白酒 12 / 朗姆+龙舌兰 6 / 跨酒种对比 4
   - 标注：每题标注正确文档 ID + 期望关键词
4. 评测脚本 `scripts/eval_retrieval.py`
   - 输入：评测集 JSONL
   - 输出：top1/top3 准确率 + 错误分析

**验收**：top1 ≥ 90%，top3 ≥ 95%

---

#### M2-02 检索质量突破（W1-W2，P0）

**目标**：从 83.3% 突破到 90%+。

**策略组合**：
1. **真实 Embedding**（M2-01 提供）
2. **低置信阈值校准**（D6 决策）
   - 跑评测集，绘制阈值-准确率-召回率曲线
   - 选择 F1 最优阈值，写入 `KB_MIN_SCORE`
3. **查询改写（Query Rewriting）**
   - 用 LLM 把用户口语化 query 改写为检索友好的关键词
   - 例：「金酒啥味」→「金酒 风味 杜松子 香料」
   - 仅当 LLM 可用时启用，Mock 时跳过
4. **HyDE 假设文档**（可选）
   - 让 LLM 先生成假设答案，用假设答案做向量检索
   - 高成本，仅在 top1 < 90% 时启用

**验收**：评测集 top1 ≥ 90%

---

#### M2-03 文档详情页（W2，P0）

**目标**：用户可查看文档全文 + 分片列表，引用跳转到具体 chunk。

**后端**：
- `GET /api/documents/{doc_id}` 返回 `{doc, chunks: [{rowid, idx, text, char_start, char_end}]}`
- `GET /api/documents/{doc_id}/raw` 返回原始内容（支持 MD/txt 下载）

**前端**：
- 新增 `DocumentDetailPanel` 组件
- 左侧：文档元信息 + 分片目录（点击跳转）
- 右侧：全文渲染（MD 渲染为 HTML，txt pre）
- chunk 区块带 `id=chunk-{rowid}`，可被引用跳转定位
- 顶部：编辑标题/标签/分类、删除、下载

**验收**：从问答点击引用 [N] → 跳转到文档详情页 → 自动滚动到 chunk → 高亮 2s

---

#### M2-04 引用跳转原文位置（W2，P1）

**目标**：M1 的滚动跳转升级为跨页面跳转。

**实现**：
- AskPanel 引用 chip 点击 → `setTab('docs')` + `setSelectedDocId(doc_id)` + `setHighlightChunk(chunk_rowid)`
- DocumentDetailPanel 接收 `highlightChunk` prop，渲染后自动滚动 + 高亮

**验收**：点击 [N] 后 1s 内定位到原文 chunk

---

#### M2-05 批量导入（W2，P1）

**目标**：支持一次上传 20 个文件。

**后端**：
- `POST /api/documents/upload-batch` 接受 multipart 多文件
- 串行解析 + 并发向量化（threadpool, max_workers=4）
- 返回 `{results: [{filename, status, doc_id?, error?}]}`

**前端**：
- 上传区支持拖拽 + 多选
- 进度条显示 "3/10 已完成"
- 完成后列表显示每篇状态（成功/失败 + 原因）

**约束**（D8 决策）：单次 ≤ 20 文件 / 100MB

**验收**：10 文件并发导入全部成功 + 进度可见

---

#### M2-06 文档标签与分类（W2，P1）

**目标**：D9 双轨制（分类单选 + 标签多选）。

**数据模型**：
- `Document` 新增 `category: str = ""`（如「烈酒」「葡萄酒」「白酒」）
- 新增 `Tag` 表：`{id, name, color}`
- 新增 `DocumentTag` 关联表：`{doc_id, tag_id}`

**API**：
- `POST /api/documents/{doc_id}/metadata` 更新分类/标签
- `GET /api/tags` 列出所有标签 + 文档数
- `GET /api/documents?category=X&tag=Y` 筛选

**前端**：
- 文档列表新增分类下拉 + 标签 chips 筛选
- 文档详情页可编辑分类/标签
- 导入时可选预设分类

**预设分类**：烈酒 / 葡萄酒 / 啤酒 / 中国白酒 / 利口酒 / 资料 / 其他

**验收**：可按分类+标签组合筛选

---

#### M2-07 历史搜索与筛选（W3，P2）

**目标**：D10 全文搜索。

**后端**：
- 新增 FTS5 表 `history_fts(query, answer)` 同步 QueryLog
- `GET /api/history?q=关键词&from=YYYY-MM-DD&to=YYYY-MM-DD&feedback=1` 支持搜索+筛选

**前端**：
- HistoryPanel 顶部搜索框 + 日期范围 + 反馈筛选
- 高亮搜索关键词

**验收**：搜索响应 < 200ms，关键词高亮

---

#### M2-08 审计日志（W3，P2）

**目标**：D12 写操作 + 问答审计。

**数据模型**：
- `AuditLog` 表：`{id, action, target_type, target_id, user, meta_json, created_at}`
- 记录：login / import / delete / seed / ask（采样 10%）

**API**：
- `GET /api/audit?limit=50&action=X` 仅管理员可查（auth_enabled 时）

**前端**：
- 新增「审计日志」Tab（仅 auth_enabled=true 时显示）

**验收**：所有写操作有审计记录

---

#### M2-09 数据导出（W3，P2）

**目标**：D11 JSON 全量 + MD 单文档。

**API**：
- `GET /api/export/all.json` 全量导出（文档+分片+历史+审计）
- `GET /api/documents/{doc_id}.md` 单文档 Markdown

**前端**：
- 文档管理 Tab 顶部「导出全部」按钮
- 文档详情页「下载 MD」按钮

**验收**：导出 JSON 可重新导入恢复

---

#### M2-10 token 用量统计（W3，P2）

**目标**：LLM token 计数 + 成本估算。

**实现**：
- `LLMResponse` 新增 `prompt_tokens` / `completion_tokens`
- `OpenAICompatBackend` 解析 `usage` 字段
- `QueryLog` 新增 `prompt_tokens` / `completion_tokens` / `cost_cny`
- 仪表盘展示累计 token + 成本

**验收**：每次问答记录 token 数

---

#### M2-11 仪表盘（W3，P2）

**目标**：基础运营指标可视化。

**指标**：
- 文档数 / 分片数 / 总字符数
- 问答数 / 今日问答数 / 平均延迟
- token 用量 / 累计成本
- 反馈分布（赞/踩）
- 准确率趋势（基于反馈）

**前端**：
- 新增「仪表盘」Tab（替代或并列历史 Tab）

**验收**：5 项核心指标实时展示
