## 1. 产品定位

### 1.1 一句话定位
**面向中文酒类爱好者与从业者的 AI 原生知识库**：用户导入酒类文档（百科/品鉴笔记/培训资料），通过自然语言问答获取带引用追溯的可靠答案。

### 1.2 核心价值主张

| 维度 | 传统搜索 | 通用 LLM | **Hermes KB** |
|---|---|---|---|
| 答案可信度 | 用户自己判断 | ❌ 幻觉严重 | ✅ 引用追溯到原文 |
| 领域专业度 | 通用搜索 | 通用知识 | ✅ 用户私有知识 |
| 中文友好 | 一般 | 一般 | ✅ 中文 BM25 + 中文 Embedding |
| 部署门槛 | N/A | API 调用 | ✅ 单进程/Docker 自托管 |
| 数据隐私 | 公开 | 上传第三方 | ✅ 自托管，数据不出域 |
| 合规 | N/A | 无未成年保护 | ✅ 年龄门 + JWT |

### 1.3 不做什么（边界）
- ❌ 不做电商/交易（不卖酒）
- ❌ 不做社区/UGC（M3 前不考虑）
- ❌ 不做通用问答（专注酒类垂直）
- ❌ 不做多租户 SaaS（M3 前单租户）
- ❌ 不做移动端原生 App（M3 前仅 Web）

### 3.2 关键用例详述

#### UC-1：爱好者快速查询（S1+S5）

**前置**：用户已导入至少 1 篇酒类文档

**主流程**：
1. 用户在问答框输入"金酒和威士忌的工艺区别"
2. 系统 50ms 内开始流式输出答案
3. 答案中 `[1][2]` 标注引用
4. 用户点击 `[1]`，跳转到对应文档的 chunk 位置
5. 用户看到原文，确认答案可信

**异常流**：
- A1：知识库为空 → 引导去导入
- A2：检索低置信 → 明确"暂无相关信息"，不编造
- A3：越狱尝试 → 拒绝并提示
- A4：LLM 超时 → 降级 Mock（用检索片段拼装）

**验收要点**：首字延迟 < 1s，引用准确率 > 95%

#### UC-2：从业者批量导入（S6+S3）

**前置**：用户有 5+ 篇 PDF/MD 文档

**主流程**：
1. 用户拖拽多个文件到上传区
2. 系统批量解析、分片、向量化、入库
3. 显示进度条 + 每篇导入状态
4. 导入完成后，用户可按标签/分类筛选
5. 用户对某篇文档点击"查看详情"，看到全文 + 分片列表

**异常流**：
- A1：PDF 解析为空 → 仍创建文档（chunk_count=0），标记警告
- A2：文件超限 → 友好提示
- A3：并发导入冲突 → WAL 重试

**验收要点**：10 篇并发导入全部成功，进度可见

#### UC-3：历史回顾与反馈（S4）

**主流程**：
1. 用户进入历史 Tab
2. 看到最近 50 条问答（含引用、模型、延迟）
3. 输入关键词搜索历史
4. 对有用答案点赞，无用答案点踩
5. 点击历史项可重新提问相似问题

**验收要点**：搜索响应 < 200ms，反馈可统计

## 5. 竞品对比

### 5.1 直接竞品

| 竞品 | 定位 | 优势 | 劣势 | 我们的差异 |
|---|---|---|---|---|
| **ChatGPT + 上传 PDF** | 通用 LLM + 文档 | 模型强、体验好 | 数据上传第三方、无引用追溯、中文酒类弱 | 自托管 + 引用溯源 + 酒类垂直 |
| **文心一言/通义千问** | 中文通用 LLM | 中文好、免费 | 无私有知识、无引用 | 私有知识库 + 引用 |
| **Notion AI** | 知识库 + AI | 协作好、生态强 | 不专注酒类、价格贵、无引用溯源 | 酒类垂直 + 引用 + 自托管 |
| **Dify/FastGPT** | 开源 RAG 平台 | 功能全、可定制 | 部署复杂、需配置、无酒类冷启动 | 开箱即用 + 酒类种子 + 合规 |

## 6. 关键产品决策

### 6.1 已确认决策（M0/M1）

| ID | 决策 | 状态 |
|---|---|---|
| D1 | LLM 接入智谱 GLM-4-Flash（OpenAI 兼容协议） | ✅ M1 已就绪 |
| D2 | Embedding 接入 bge-small-zh-v1.5（SentenceTransformer） | ✅ M1 已就绪 |
| D3 | 单用户 JWT 认证（非多租户） | ✅ M1 已实现 |
| D4 | M3 初期迁移 Postgres + pgvector | ⏳ M3 |
| D5 | M3 后定商业化（订阅/Api Key） | ⏳ M3 |

### 6.2 M2 新增决策点

| ID | 决策点 | 倾向方案 | 决策依据 |
|---|---|---|---|
| **D6** | 检索准确率提升路径 | 真实 Embedding + 阈值校准 + 查询改写 | M1 已证明 Hash 84% 已达上限 |
| **D7** | 文档详情页形态 | 单页 SPA，左侧目录 + 右侧全文，chunk 高亮 | 用户校验原文核心场景 |
| **D8** | 批量导入上限 | 单次 ≤ 20 文件 / 100MB，超出提示分批 | 防止 OOM + 友好提示 |
| **D9** | 标签 vs 分类 | 双轨：分类（单选）+ 标签（多选） | 分类主组织，标签辅助检索 |
| **D10** | 历史搜索范围 | 全文搜索（query + answer + citations） | 用户记不住原问题 |
| **D11** | 数据导出格式 | JSON 全量 + Markdown 单文档 | JSON 备份，MD 分享 |
| **D12** | 审计日志粒度 | 写操作（导入/删除/登录）+ 问答 | 合规 + 成本核算 |

## 7. 非功能性需求

### 7.1 性能

| 指标 | M1 实测 | M2 目标 | M3 目标 |
|---|---|---|---|
| 流式首字延迟 | < 50ms（Mock） | < 800ms（真实 LLM） | < 500ms |
| 非流式问答 | 200ms | 1.5s | 1s |
| 检索准确率 top1 | 83.3% | ≥ 90% | ≥ 95% |
| 10 并发导入 | 100% | 100% | 50 并发 100% |
| 历史搜索 | N/A | < 200ms | < 100ms |
| 文档详情页加载 | N/A | < 500ms | < 200ms |

### 7.2 可用性

- 单进程部署，重启 < 5s
- 数据库自动备份（M2 增加定时导出）
- 降级策略：LLM/Embedding 异常时 Mock/Hash 兜底

### 7.3 安全

- 越狱检测 + 输出泄露检测（M1 已有）
- JWT 认证（M1 已有）
- 操作审计（M2 新增）
- 数据加密 at-rest（M3 考虑）

### 7.4 合规

- 未成年保护（M1 已有）
- 数据不出域（自托管保证）
- 酒类内容免责声明（M2 在年龄门增加）

## 9. 风险与依赖

### 9.1 技术风险

| 风险 | 概率 | 影响 | 缓解 |
|---|---|---|---|
| 真实 Embedding 维度与 Hash 不一致需重建索引 | 高 | 中 | M2 提供 reindex API |
| LLM API 限流/超时 | 中 | 高 | 重试 + 降级 Mock |
| 大 PDF 解析 OOM | 中 | 中 | 流式解析 + 文件大小限制 |
| SQLite 并发瓶颈 | 低 | 中 | M3 迁 Postgres |

### 9.2 业务风险

| 风险 | 缓解 |
|---|---|
| 用户不愿导入文档 | 提供 5 篇种子 + 一键导入酒类百科 |
| 检索准确率不达预期 | M2 评测集 + 阈值校准 + 查询改写 |
| 未成年保护合规争议 | 年龄门 + 免责声明 + 不卖酒 |

### 9.3 外部依赖

| 依赖 | 用途 | 风险 |
|---|---|---|
| 智谱 BigModel | LLM | 限流/价格/服务中断 |
| bge-small-zh-v1.5 | Embedding | 模型下载/版本变更 |
| HuggingFace | 模型分发 | 网络访问 |
| pypdf | PDF 解析 | 复杂 PDF 兼容性 |
